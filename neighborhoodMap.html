Neighborhood Map

Neighborhood Map shows the Montreal city featuring some turistic locations. By clicking a location pin, a popup opens to give the user more information about the selected place (data from Wikipedia API). The map also allows the user to browse the content in different ways.

Project Purpose and Goal

The Neighborhood Map was the last project from the Front-End Web Developer Nanodegree program at Udacity. It was a React project and we had to build it from scratch. We were not given any initial code to work on, only the guidelines on how to complete the project.

The goal of this project was to develop a single page application featuring a map of a neighborhood and adding third-party data to give us more information about these locations.

Once the map is loaded, the user will see a list of the touristic places in Montreal on the left and a map of Montreal with the red location pins on the right side. For the map I have used the google-maps-react library.

When a location pin is clicked, the color of the pin will change to blue and some information regarding that location will show up with a link to the full Wikipedia article.

The user can also search a location by typing its name in the search box above the location list. It will remove all other location pins but the one searched by the user.

Another way to browse through the content is by clicking at an item in the location list. Clicking a location in the list will change its pin color to blue and show the information regarding that location.

Problems and Thought Process

The Neighborhood Map was the first project that I did from scratch, without the support of any initial code. The first challenge I had was to think about the code structure and design, figuring out which components I needed to create and the props I had to pass from one to another. After that was deciding which React Google Maps library to use.

I decided to go with google-maps-react after reading its documentation and checking its latest updates.

Lessons Learned

By working on this project, I have learned how to start a project from scratch. Thinking about the structure and design before jumping into the code. I have also learned what to check before choosing a component's library, reading through API and library documentations and testing API calls via POSTMAN.

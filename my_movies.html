My Movies

My Movies is a project that I created to improve my understanding of the foundation and core concepts of React. In this project, I used the New York Times API to gather information about the movies' review data and injected this data based on the user's input of date range.

Project Purpose and Goal

After completing the Front-end development nanodegree from Udacity and the React course from TylerMcginnis.com. I have decided to create a React project on my own that was unique and where I could practice the React knowledge I have gained in those courses.

The goal of this project was to create an application where the user can track the movies he/she watches. The list of movies is based on the critics' recommendation of the New York Times.

Despite being the first page of the application, the Login component was one of the last features I have added to the project. I have created the login page to better understand Javascript promises and learn how to simulate an API call using promises.

After entering the login credentials and clicking on the submit button, a spinner is shown while trying to login. If the login fails a message will be displayed to the user.

For the spinner, I have used the react-spinner-material library.

After a successful login, the user is taken to the search page. In this page, the user can enter the period of the movie review.

If there is no reviews in period, a message will be displayed to inform the user.

After the search is completed, up to 20 movies picked by the critics (according to the New York Times API) will be loaded to the 'All' tab. Working with the NYT API gave me more experience on reading API documentations and practice making calls through POSTMAN.

Once all movies are loaded, the user can change its status by clicking the blue down arrow located on the top left of each movie card and choosing either 'Wish List' or 'Watched'.

In this part of the project, I practiced object manipulation by changing the movie status (status is a property of the movie object) and some ES6 syntax like the spread operator.

The movies changed to wish list will show up in the 'Wish List' tab with a heart icon on the top right of the card.

Once the movie status is changed to watched, the user is given the option to give his/her own rate to the movie. In the 'Watched' tab, all movies will have a green checked icon on the top right of the card.

The user can give a rate from 1 to 5 and it will be converted in stars. If the user tries to give a rate above 5, an alert message will be shown. For the stars and all other icons, I have used the react-icons library.

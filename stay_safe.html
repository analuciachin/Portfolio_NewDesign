Stay Safe

Stay Safe is a web application to help people book an appointment to get their Covid-19 vaccine. It was built with 2 types of user in mind: Patient and Nurse. If the user logs in as a patient, he can book, update and delete an appointment. A new patient can sign-up, after signing up, he fills out a profile form to determine if is a high priority patient to get vaccinated. If the user logs in as a nurse, he can see a list of all the patients who booked an appointment with him.

Project Purpose and Goal

I came up with the project's idea after booking an appointment to get a flu shot. The application was good, but I found some flaws and issues and wanted to build an application where I could address those problems. For instance, one of the issues was that after booking an appointment if the user wanted to reschedule it or change the clinic, there was no way to do it through the app. The user would need to call the clinic and ask to have the appointment updated.

By working on this project I also got a better understanding of REST API and back-end. The back-end of this project was built using NodeJS, Express and PostgreSQL.

The login is also the home page. In the login page the user can choose whether to log in as a nurse or a patient. As a nurse the user can log in using the username/password and as a patient the user can log in using his email/password. A user can also sign up as a new patient.

After logging in, if a patient has an appointment booked, he has the option to either update or delete the appointment. If the user chooses to update the appointment, he needs to fill out a form with the new appointment information. Once the appointment is updated, the user gets a confirmation message.

If a user does not have an appointment booked or if the appointment was deleted, he can fill out the form to book a new appointment

After a new patient signs up, he needs to fill out a form with his profile information to determine if he is in the high priority group to get vaccinated. After filling out the form, he gets redirected to a page where he can book an appointment.

If the user logs in as a nurse, he can see a list of all the appointments booked with him.

Problems and Thought Process

Building the back-end was the most challeging and rewarding part of this project. Understading the best practices of creating REST API endpoints and working with working with chained queries and using promises to perform sequential database queries.

Web Stack and Explanation

Throughout my journey of learning React, I have used a few public APIs to get data for my projects. It was a mystery to me how I could create my own API and I was eager to learn it. When I planned Stay Safe, I decided I wanted to work on both sides, front-end and back-end. The back-end was built using NodeJS, Express and PostgreSQL. Working on the back-end gave me a deeper understanding of REST API and how the back-end communicates with the database.

The front-end was built using React and Bootstrap to help with the styling. I already have some knowledge of the React using Class Components, so for this project I have decided to use Functional Components and Hooks instead.

Lessons Learned

By building Stay Safe, I got a better understanding of REST API and developing a back-end using NodeJS, Express and PostgreSQL. It also helped me to better understand React Hooks (useState and useEffect).

Future Improvements

The next steps for this project would be to add hash passwords, set different date ranges for booking an appointment considering the patient's priority, making the application responsive to all screen sizes, improving the UI and refactoring the code to make it as clean and reusable as possible.
